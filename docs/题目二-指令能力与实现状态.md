## 4. 预先识别的风险（设计期判断，可能成为"未完成原因"）

按风险从高到低，提前写明取舍预案，避免提交时原因说明空泛：

1. **#16/#26 创作拆解效果不可控**（LLM 生成的组合图形部件错位、比例失调）
   → 预案：内置 4 个手工调优的组合模板（雪人/房子/笑脸/太阳）兜底走规则层，LLM 自由拆解作为加分演示；若 LLM 效果差，#26 降级或放弃，原因记为"生成质量不达演示标准，模板兜底已覆盖核心评分点"。
2. **#19 投机解析**依赖 ASR partial 的稳定性，若火山 ASR partial 抖动大（频繁修正前文），投机命中率低收益有限
   → 预案：埋点统计 partial==final 比例，< 50% 则关闭该特性，原因记为"ASR partial 修正率过高，投机收益不抵复杂度"。
3. **#18 澄清多轮**涉及状态机子态与超时窗口，联调成本高
   → 预案：时间不足时降级为"播报歧义 + 用户重说完整指令"，不做定向追问。
4. **#10 同音词纠错**词表覆盖度问题（长尾口音/方言）
   → 预案：只保证绘图领域核心词表（约 60 词）；通用纠错明确不做，依赖 ASR alternatives 给 LLM 参考。
5. **云端 ASR（火山）/TTS 接入阻塞**（配额/文档/网络）
   → 预案：Provider 抽象已设计热切换 WebSpeech；若发生，演示用 WebSpeech，原因如实记录并说明协议层未受影响。
