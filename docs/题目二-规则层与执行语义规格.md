### 2.4 量词与尺寸映射（数值唯一来源）

**移动距离（`move.delta` 的标量）**

| 话术 | 像素 |
|---|---|
| 一点 / 一点点 / 稍微 | 60 |
| 一些 / 一段 | 120 |
| 很多 / 一大截 / 远一点 | 240 |
| 显式数字（"100"/"一百"） | 原值（中文数字转阿拉伯） |

**缩放（`resize.scale`）**

| 话术 | factor |
|---|---|
| 大一点 / 变大 / 大一些 | 1.3 |
| 大很多 / 大大的 | 1.8 |
| 大一倍 / 放大一倍 / 两倍 | 2.0 |
| 小一点 / 变小 / 小一些 | 0.77 |
| 小很多 | 0.55 |
| 缩小一半 / 一半 | 0.5 |

**语义尺寸 → 特征尺寸 v（`SizeSpec` 的 "small"/"medium"/"large"）**

| | small | medium（缺省） | large |
|---|---|---|---|
| v | 40 | 80 | 160 |

触发词：小/小小的→small；大/大大的/很大→large；迷你→small；巨大→large×1.5（=240）。

**特征尺寸 v → 各形状几何（引擎换算规则）**

| shape | 换算 |
|---|---|
| circle | 半径 = v |
| ellipse | 长半轴 = v，短半轴 = 0.6v |
| rect（正方形） | 边 = 2v |
| rect（长方形） | 宽 = 2v，高 = 1.5v |
| triangle | 边长 = 2v |
| star | 外半径 = v，内半径 = 0.5v |
| line | 长度 = 3v |
| text | fontSize = max(16, 0.5v) |

**相对尺寸**："比 X 矮/小一点" → `{relativeTo: X, factor: 0.7}`；"和 X 一样大" → factor 1.0；"比 X 大/高" → 1.3。
维度规则：用于 `width` 槽位 → factor × 参照包围盒宽；`height` → factor × 参照高；`size`（特征尺寸）→ factor × max(参照宽, 高) / 2。

**间距**：`gap` 缺省 20（对象间外贴）；ref=canvas 时内边距 40；对象内贴（`inside`）时 gap 作内边距、缺省 0。

**缺省样式**（创建时未指定颜色）：闭合图形 fill `#2D7DD2`（v1.7 由暗灰 #4B5563 换饱和蓝，暖纸上更出彩）；line/polyline stroke `#111827`、strokeWidth 4（v1.7 由 3 加粗）；text fill `#111827`。rect 仅给 size 时默认长方形（宽 2v 高 1.5v），正方形由理解层显式产出 width=height。

### 2.5 忽略词（匹配前剔除，不计入未消费 token）

帮我、请、给我、麻烦、来、一下、吧、啊、呢、哦、嗯、那个（句中填充）、然后、就、把（保留语法功能后剔除）。

### 2.6 确认窗口词表（AWAIT_CONFIRM 子态专用）

- 肯定：确认、确定、对、对的、是、是的、好、好的、嗯、可以、行、清空吧、删吧
- 否定：取消、不、不要、不用、别、算了、等等
- 匹配规则（v1.7 修）：对转写做**包含**匹配而非整句精确相等——「我确认 / 确认清空 / 确定吧」等带前后缀口语都算肯定（旧版整句精确相等导致这类确认被误判取消）；**否定优先**：含否定词即取消（即使同时含肯定词，如「不用确认了」）；同时对纠错后文本再判一次以兜住语音同音偏差。
- 含肯定词且无否定词 → 执行；其他任何转写（含纯否定/无法匹配）→ 视为否定（保守策略），TTS"已取消"

### 2.7 序数与指代

第一个/第1个→`ordinal:1`，第二个→2……；最后/最后一个/最新的→`"last"`；最早/最先→`"first"`。
它 / 这个 / 那个（句首单独成分）/ 刚才那个 / 刚画的 → `{byFocus:true}`。

## 3. 同音词纠错

### 3.1 流程

```
ASR final → ① 精确词表替换（§3.2，带上下文条件） → ② 剩余未识别 token 拼音回退（§3.3） → 进路由器
```

纠错只发生在规则层匹配之前；送 LLM 的 `utterance` 用纠错后文本，原文放 `asr_alternatives[0]`。

> 词表维护：开发期遇到新 badcase 直接加表；"保持"条目用于测试基线，确认不会被误纠。

### 3.3 拼音回退（词表未命中时）

1. 对纠错后仍无法参与任何模板匹配的 2~3 字 token，求其无声调拼音
2. 与领域词典（§2.1~§2.3 全部词条 + 动词表）逐一比对拼音编辑距离
3. 距离 ≤ 1 且唯一命中 → 替换；多个命中或距离 > 1 → 不替换（留给 LLM）
4. 实现：`pinyin-pro` 库；词典预计算拼音建索引，单次纠错 < 5ms

## 4. 规则模板文法

> **现状（refactor·去绘图规则模板）**：规则快路径已瘦身，**只保留 LLM 无法产出的系统指令**——
> T6 撤销/重做、T7 清空（confirm-pending）、T10 保存图片。**绘图/编辑模板 T1-T5/T8/T9/T11
> 已移除**，相应口语（创建/移动/缩放/改色/删除/旋转/命名/选中）一律升 LLM（边生成边画、看
> 绘制过程）。下表 T1-T11 保留作历史/语义参照；§2 词表、§3 纠错、分词机器与 §4.3 mode 判定
> （parse/plan）仍在用（decideMode/extractPlanSubject 依赖分词）。系统指令同义词已从宽扩充
> （无 LLM 兜底，宁可宽），见 §2 VERB_WORDS。

### 4.1 匹配流程

```
纠错后文本
 → 剔除忽略词（§2.5）
 → 分词 + 槽位标注（最长匹配扫描 §2 各词表，产出 token 流：
    [动词][颜色][尺寸][形状][方位][数量][目标指代][名称]…）
 → 按 T1~T11 依序尝试模板
 → 命中且 未消费 token 占比 ≤ 30% → ParseResult{source:"rule", confidence:1.0}
 → 否则 → 升级 LLM（mode 判定见 §4.3）
```

“未消费 token 占比”按字符数计。阈值 30% 的含义：允许少量口语噪音，但句中存在成段未理解内容时宁可升级 LLM 也不猜。

### 4.2 模板清单 T1~T11

| # | 模板（槽位顺序宽松，`?`=可选） | 产出 Op | 示例 |
|---|---|---|---|
| T1 创建 | 动词(画/添加/加/来/写) + 位置? + 尺寸? + 颜色? + **形状** + 内容?(text 用) + 位置? | `create` | "在左上角画个大的蓝色矩形" |
| T2 移动 | 目标? + 动词(移/挪/移动/拖) + **方向**(上/下/左/右) + 量词? | `move.delta`（方向×量词，缺省 60） | "把圆往右移一点" |
| T3 缩放 | 目标? + (变/放/缩) + **大/小** + 量词? ｜ 放大/缩小 + 倍数? | `resize.scale` | "把它变大一点" |
| T4 改色 | 目标? + (涂成/变成/改成/换成/涂) + **颜色** | `style.fill` | "把房子涂成红色" |
| T5 删除 | **目标** + (删掉/删除/去掉/移除/擦掉) | `delete` | "把那个圆删掉" |
| T6 撤销/重做 | **撤销/撤回/回退** + N步? ｜ **重做/恢复** | `undo/redo` | "撤销三步" |
| T7 清空 | **清空**(画布)? ｜ 全部删掉 ｜ 重新开始/重画 | `clear` → **confirm-pending** | "清空画布" |
| T8 旋转 | 目标? + (转/旋转) + **N 度** ｜ +方向(顺/逆时针)? | `rotate.degrees`（逆=负） | "把三角形转 45 度" |
| T9 命名 | (这个/它) + **叫/命名为** + 名称 | `rename`（目标=焦点） | "这个叫屋顶" |
| T10 导出 | **保存/导出/下载** + 图片? | `export png` | "保存图片" |
| T11 选中 | **选中/选择** + 目标 | `focus` | "选中那个红色的圆" |

**目标槽位的统一解析**（T2~T5、T8、T11 共用）：
1. 指代词（§2.7）→ `byFocus`
2. 已命名对象名 → `byName`（名称表来自场景图，匹配时热更新）
3. `[颜色?] [序数?] [形状]` → `byQuery`
4. 槽位缺失（"变大一点"没说目标）→ `byFocus`；若焦点为空 → 不命中，**不升级 LLM**，直接 TTS"请先告诉我要操作哪个图形"（这是明确的信息缺失，LLM 也解决不了）

**T1 细则**：
- 位置槽位 = 方位词（→ `{ref:"canvas", anchor}`）或相对介词结构"在 X 的 D 边"（→ `{ref:目标, anchor:D}`）。两处位置槽位最多出现一个。
- "画两个圆" → 数量词展开为 2 个 `create`（自动布局错开），上限 5 个，超出升级 LLM。
- text 形状必须有引号内容或"写/字是"引导（"写上你好"→ `text:"你好"`），否则不命中。

### 4.3 升级 LLM 时的 mode 判定

命中以下任一 → `mode:"plan"`，否则 `"parse"`：
- 动词"画"+ 非 §2.2 词表的具体名词（雪人/房子/树/太阳/小猫…即"画的不是基础图形"）
- "一幅 / 一张 / 一个场景 / 风景"
- 句含 2 个以上"然后/再/接着"连接的创建动作

## 5. 执行引擎语义细则

### 5.1 焦点对象（focus）更新规则

焦点含 id 与**粒度 focusScope**（'group' | 'object'，v1.1）。粒度决定 `byFocus`（"它"）的作用范围：

| 事件 | 焦点 id | focusScope |
|---|---|---|
| 单个 `create`（parse/规则） | = 新对象 | object |
| `llm-plan` 自动编组 / `group` op | = 最后成员 | **group**（"它"=整组） |
| 几何类 `move/resize/rotate/zorder` 作用整组（组名/byFocus-group） | = 目标 | group |
| 几何类作用单成员、`style/setText/rename` | = 被操作对象 | **object**（"它"=该部件） |
| `delete` / `clear` | 清空 | 清空 |
| `focus` op | 显式设置 | object |
| `undo/redo` | 恢复快照中的焦点（含粒度，见 §5.4） | 同快照 |

**byFocus 的组/部件二义性（v1.1 核心修订）**：`byFocus` 几何操作仅在 `focusScope='group'` 时提升整组（"画完猫→把它移走"=整组）；编辑过某部件后粒度降为 `object`，此时"把它再大点"只作用该部件，不再误伤整只。理解层据 `scene.focus.scope` 决定"它"的指代，并据 `scene.groups` 精确引用部件名（避免误用 byFocus/组名动整组）。

### 5.2 自动布局（create 缺省 `at` 时）

确定性算法（无随机，保证可测试）：

```
1. 画布为空 → center (512, 384)
2. 有焦点对象 F → 依次尝试 F 的 right / below / left / above（gap=40），
   取第一个与所有现有 bbox 不重叠、且完全在画布内的位置
3. 仍无 → 依次尝试画布九宫格锚点（center, top-left, top, top-right, left,
   right, bottom-left, bottom, bottom-right，内边距 40），同样判重叠
4. 仍无（画布很满）→ 直接 center 叠放，不报错
```

### 5.3 相对定位的精确计算

记 ref 包围盒 `R`，新对象包围盒 `N`（尺寸已由 SizeSpec 确定）。

**ref 为对象 → 外贴（outside）**：

| anchor | N 的位置 |
|---|---|
| `left` | N.right = R.left − gap；N 垂直中心 = R 垂直中心 |
| `right` | N.left = R.right + gap；垂直居中对齐 |
| `top` | N.bottom = R.top − gap；水平居中对齐 |
| `bottom` | N.top = R.bottom + gap；水平居中对齐 |
| 四角（如 `top-left`） | N 的 bottom-right 角 = R 的 top-left 角 + (−gap, −gap) |
| `center` | N 中心 = R 中心（叠放，画雪人眼睛等场景） |

**ref 为 canvas → 内贴（inside）**：N 贴画布对应边/角，内边距 40；`center` = 画布中心。

**ref 为对象且 `inside: true` → 内贴**：N 贴 R 对应边/角的**内侧**（如 `bottom` → N.bottom = R.bottom − gap，水平居中），gap 作内边距、缺省 0（贴齐参照边缘）。用于"门嵌在房子底边""窗在房子内部"等部件内嵌场景——外贴语义表达不了这类位置（v1.1 新增）。

**line/polyline 显式 points + ref 为对象 → 首端点贴合（v1.2）**：锚定 `points[0]` 到 R 对应边/角的锚点（gap 沿 anchor 方向向外、缺省 0）——"手臂长在身体上"类连接部件，包围盒贴合永远贴不到线的端点。无显式 points（引擎默认横/竖线）维持包围盒外贴。

**`onEdge: true` → 边缘贴附（v1.3，v1.3.1 修订咬合度）**：锚点 = R **真实形状边缘**与 anchor 方向射线的交点（circle 圆周 / ellipse 参数化 / 其余 bbox 周界近似），gap 沿方向外移缺省 0。线类钉 points 首点；其余部件**骑在边上**：沿方向 1/3 咬合、2/3 外露（初版钉中心=半埋，实测"耳朵长进头里、脚陷进身体"）。用于猫耳贴头、帽子贴头顶、轮子贴车底——依据：bbox 锚定在圆形参照的斜向方位悬空（耳朵距头心 127px > 半径 97.5，多轮修复发散的实测根因）。

`offset` 在上述结果上叠加。最终位置再过 §5.5 clamp。

### 5.4 undo/redo：快照制

- 不做逆操作（command inverse），**每事务结束后保存整幅场景 JSON 快照**（含焦点 id）。画布对象数量级 ≤ 百，快照成本可忽略，换取绝对正确性
- 栈深 50，超出丢最旧；redo 栈在新事务提交时清空
- `undo steps:n` = 弹 n 帧；n 超过可用深度时撤到栈底为止、不报错（栈本来就空才报 NOTHING_TO_UNDO）
- `clear` 也入栈（可被撤销——这是清空敢于轻确认的底气）；clear 不重置 id 计数（保证 id 全程不复用）
- 事务部分失败（executed > 0）时同样入栈：已生效的部分必须可撤销

### 5.5 越界 clamp

1. bbox 超出画布 → 平移最小距离使其完全入界
2. 对象本身大于画布（平移无解）→ 等比缩小至画布的 90% 后居中
3. clamp 发生时不报错、不播报，仅事件总线发 `exec.clamped`（调试面板可见）

### 5.6 group 的引用语义

- **几何类 Op**（move/resize/rotate/delete/zorder）的组提升（v1.1 修订）：仅当意图明确指组时提升整组——
  target 为**组名引用**（byName=组名，"把雪人移到右边"）或**焦点指代**（byFocus，"把它移走"）；
  显式命中成员（成员名/byId/byQuery）则**作用成员本身**（"把头变大一点"只放大头——
  原"命中成员即提升"语义使部件级多轮修正无法进行，实测修复成功率 2/5，故修订）
- **外观类 Op**（style/setText/rename）：作用于**成员本身**（"把雪人的鼻子涂红"改的是鼻子）
- byQuery 在组内外都有命中时：优先组外独立对象；仅组内命中则按上述规则
- `ungroup` 后成员保留各自 id 与 name；组名作废

### 5.7 歧义处理（AMBIGUOUS_TARGET）

- 候选 ≤ 3：TTS 列举区分特征——优先用颜色区分（"有红色和蓝色两个圆，要哪个？"），颜色相同用位置（"左边的还是右边的？"），并把候选写入 `clarify.expecting`
- 候选 > 3：TTS"画布上有 N 个圆，请说得具体一点，比如颜色或位置"
- 澄清回答的快匹配：下一句 final 先与 `expecting` 做包含匹配，命中则直接补全原 Op 执行，**不再走完整解析**

### 5.8 表达力扩展（v1.5）

为消除 LLM 手算几何关系的高频出错（对称偏移、对齐、相对层级），把这些计算下沉为引擎原语：

- **`mirror`**：在 `target` 关于 `about` 中心轴的镜像位置复制一个新对象（含几何与 points 翻转、rotation 取负）。`axis` 缺省 `vertical`（左右镜像）。对称部件（双耳/双眼/双臂）的标准做法——右件 = 左件镜像，避免两组手算 offset 不对称。新对象不继承组（编组由 autoGroup/调用方决定）。
- **`radial`**（v2.x GH #111）：把 `target` 绕 `about` 中心等角复制成 `count` 份（含原件）——引擎补 `count-1` 个旋转副本，步长 `360/count` 度。`about` 可为 TargetSelector（取其中心）或显式 `{x,y}` 坐标。vpath 类对象走 `rotatePathD`（坐标成对旋转）；图元走中心绕转 + `rotation` 跟随。用途：花瓣/光芒/轮辐/雪花臂/齿轮齿——`mirror` 的 N 重泛化，引擎精确均布，胜过 LLM 手画一圈不齐。焦点留在原件。
- **`align`**：`targets` 中除首个外，全部在 `axis` 上的中心对齐到首个目标的中心。
- **`distribute`**：`targets` 按 `axis` 排序后首尾不动，中间对象中心等距分布（≥3 个）。`align` + `distribute` 组合处理"一排纽扣/窗格/装饰球"，免手算每个坐标。
- **`between` 位置**：新对象中心落在两参照中心连线的 `t`（缺省 0.5）插值点（"脖子在头和身体之间"），`offset` 叠加。
- **连接线 `from`/`to`**（仅 `line`）：端点各自落在 from/to 对象**真实形状边缘**上、朝向彼此的点（复用 onEdge 的边缘求交），与 `points`/`at` 互斥——"两物之间连一条线"无需手算端点。
- **`zorder` 相对层级**：`to:{above:目标}` / `{below:目标}` 把对象层级落在参照的紧邻上/下方（分数 z，渲染按 z 排序），表达"把太阳放到云后面"。

### 5.9 几何接地与持久锚定（v2.3 GH #119）

`attach` op 把"相对关系"从解析即抛弃改为**持久化到 `SceneObject.anchor` 字段**：

- **建立锚定**：发全参 `{op:'attach', target, to, parentAnchor, mode, childAnchor?, offset?, gap?}` → 把关系写进 `target.anchor`，并按父件当前真实几何立即解析，平移 target（vpath 用 x,y 平移使 childAnchor 落在父件 onEdge 点，`d` 不变）。
- **重解析（reanchor）**：仅给 `{op:'attach', target}`（无 to 字段）→ 按 `target.anchor` 已存关系 + 父件当前几何重贴（多轮修正"手画偏了"直接 reanchor，无需 LLM 手算坐标）。
- **持久化**：`SceneObject.anchor` 随 scene 进 localStorage；老存档无此字段 → 视为无锚定，优雅降级。
- **undo/redo**：history 是 scene 快照，anchor 字段自动随快照进退，无需特判。
- **公用 resolver**：`resolveAnchorPoint` 抽纯函数，create 定位与 attach 重解析共用同一套几何（`edgeAnchorPoint` / `resolvePosition`），杜绝两套漂移。
- **`forwardRetry` 兼容**：attach 引用未创建父件 → 走现有 TARGET_NOT_FOUND 暂存重试，无需特殊处理。

