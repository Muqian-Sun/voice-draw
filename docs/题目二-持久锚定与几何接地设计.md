# 持久锚定与几何接地设计（RFC）

> 状态：**设计 / 待实现**（2026-06-14 提出）。本文为前瞻 RFC；Phase 1 落地后，其执行语义并回《规则层与执行语义规格》§5.3。

## 1. 动机

多轮编辑实测（「画白雪公主和七个小矮人」）暴露：部件（手/脚）创建时位置就偏、且**多轮语音修正改不动**。根因有两层，均已从代码核实：

- **关系算完即弃**：解释器把 `at:{ref,anchor,onEdge,between}` 解析成一个绝对点（interpreter.ts:365-406）后只把结果写进对象的 `x,y` 平移位；`SceneObject`（engine/scene.ts）**无任何关系字段**（parent/anchor）。于是"手贴在袖口"这件事没人记——修正时 LLM 只能拿 bbox/center 重新盲算。
- **LLM 不擅长坐标几何**：Transformer 无空间表征，多部件 + 相对真实形状 + 看不见画面的像素落点必然累积误差；但它**擅长空间语义**（知道手贴袖口、`公主左手`↔`公主左袖`）。日志即证：命名全对（语义成功），坐标全偏（几何失败）。

**关键事实**：引擎其实**有**几何接地能力——`onEdge`（edgePointInDir，按真实边缘求点）、`between`、`placeOutside/Inside`，且 `positionSchema` 被 `at` 与 `move.to` 共用（schema.ts:212），从修正也够得着。问题不是"引擎不会算几何"，而是**算完把关系扔了**。

## 2. 原则

**引擎管几何，LLM 管意图。** LLM 只声明关系（"手锚在袖口 onEdge"），引擎解析**并持久化**，任何变化按父件当前真实几何**再解析**。设计贴着 LLM 能力边界：最大化引擎算、最小化 LLM 算。

## 3. 数据模型

`SceneObject` 增可选字段：

```ts
anchor?: {
  to: TargetRef            // 父件（byName/byId）
  parentAnchor: Anchor     // 贴父件哪条边（如 bottom）
  mode: 'onEdge' | 'outside' | 'inside' | 'between'
  childAnchor?: Anchor     // 子件自身哪个点去贴（缺省 center；手用 top=手腕侧）
  offset?: [number, number]
  gap?: number
}
```

- **持久化**：随 scene 进 localStorage；老存档无此字段 → 视为无锚定（优雅降级，同 `safeColor` 模式）。
- **undo/redo**：history 是 scene 快照，字段自动随快照进退，无需特判。

## 4. resolver 抽取

把 interpreter.ts 内 `at→point` 段抽成**纯函数** `resolveAnchorPoint(scene, spec) → Point`（复用 edgePointInDir / placeOutside / between）。创建定位与再解析**共用同一份**，杜绝两套漂移。纯重构、无行为变化、可独立成 PR。

## 5. Op 设计

合并为单个 `attach` op（减词表）：

- **建立 + 解析**：给全参 `{op:'attach', target, to, parentAnchor, mode, childAnchor?, offset?, gap?}` → 把关系写进 `target.anchor`，并立即按父件真实几何解析、平移 target（对 vpath = 设 `o.x/o.y` 让其 childAnchor 落在父件 onEdge 点；vpath 坐标烤在 `d` 里，靠平移位归位、无需重写 `d`）。
- **再解析（reanchor）**：只给 `{op:'attach', target}` → 按 `target.anchor` 已存关系、父件当前几何重贴。

修正话术映射："手画偏了" → 有 anchor 则 reanchor；无（既有部件绝对坐标放偏、无关系）则模型按名字推父件发完整 attach（`公主左手`→`公主左袖` bottom onEdge）建立并归位。

## 6. 分阶段（每阶段 = 若干小 PR，合赛制）

- **Phase 1（核心）**：① resolver 抽取（重构 PR）② `SceneObject.anchor` 字段 + `attach` op（schema + interpreter + 引擎/ golden 测试）③ Prompt 配套。修正"手画偏了"即可用。
- **Phase 2（rig）**：父件 move/resize/reshape → **级联重贴**子件（children 索引、拓扑序、防环）。group-move 已整组平移、相对位置不变，**不在此列**。
- **Phase 3（感知）**：scene 上下文回填每对象的 `anchor` + 关键边缘锚点，让 LLM 选对父件/边。

## 7. 与现有机制交互（逐一核实）

| 机制 | 影响 |
|---|---|
| group-move | 整组同 Δ 平移，锚定子件天然跟随，无需 reanchor ✅ |
| forwardRetry 流式 | attach 引用未创建父件 → 走现有 `TARGET_NOT_FOUND` 暂存重试 ✅ 复用 |
| 持久化 / undo | 快照天然支持；老存档无字段优雅降级 ✅ |
| autoGroup / 焦点 | 不受影响 |

## 8. Prompt 配套（Phase 1 同期）

- 连接件（手/脚/耳/帽/轮）**创建时优先 `at:{ref,onEdge}` 或 attach**，别把绝对坐标烤进 `d`。
- 修正话术（"X 画偏了/位置不对"）→ **attach 到逻辑父件**（手→袖 bottom onEdge、脚→腿），禁止盲算 delta。
- （Phase 3）scene 上下文回填 `anchor` 让"它该在哪"可读。

## 9. 风险 / 待定

- **childAnchor 粒度**：v1 用 bbox 锚点近似（手 top 贴袖 bottom）；更精细（具体路径点）留后。
- **reshape 父件**：vpath 改 `d` 后子件需按新边缘重贴——reshape 也触发级联（Phase 2）。
- **环 / 链**：A 锚 B、B 锚 A；Phase 2 拓扑排序 + 检测。

## 10. 测试

- 引擎单测：attach 把 vpath 手平移到袖真实下沿；父件 move → reanchor 跟随；各 mode；老存档无字段不崩。
- golden：「手画偏了 → 接回袖口」端到端（构造 op，不依赖 LLM 随机性）。

## 11. 与"按角色拆子计划"（瓶颈⑤）的关系

一对互补：

- **子计划⑤**：保证**创建时**关系对（每个角色在局部小坐标系内画，手天生贴袖、模型算的范围小误差小）。
- **持久锚定**：保证**之后改得动 + 父件变了不脱节**。

**次序**：先锚定 Phase 1（直接修当前"改不正"痛点、独立可用）→ 再视情况上子计划⑤。
