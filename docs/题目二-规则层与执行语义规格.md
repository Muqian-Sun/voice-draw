# 题目二：规则层与执行语义规格（v1.0）

本文档是**三方共用的唯一来源（Single Source of Truth）**：

| 使用方 | 使用本文档的哪部分 |
|--------|------------------|
| 规则快路径（本地解析器） | §2 词表、§3 纠错、§4 模板文法 |
| LLM System Prompt | §2 词表中的量词/颜色/尺寸映射（附录 A 引用同一数值） |
| 绘图执行引擎 | §5 执行语义细则 |
| 测试 | 附录 B Golden 测试集 |

**一致性规则：修改 §2 的任何映射数值，必须同步更新附录 A 的 System Prompt 对应行，并跑通附录 B。**
实现上，§2 的全部表落地为 `src/shared/lexicon.ts` 一个模块，规则层 import、System Prompt 由其生成（保证逐字节稳定：构建期生成，非运行期拼接）。

### 2.3 方位词 → Anchor

左上/左上角→`top-left`，右上/右上角→`top-right`，左下→`bottom-left`，右下→`bottom-right`，
中间/中央/正中/中心→`center`，上面/上方/顶部/上边→`top`，下面/下方/底部/下边→`bottom`，
左边/左侧/左面→`left`，右边/右侧/右面→`right`。

相对方位介词（触发 `ref` 为对象而非 canvas）："在 X 的左边/右边/上面/下面/旁边"。"旁边" → `right`（若右侧空间不足 80px 则引擎自动改 `left`）。

### 2.4 量词与尺寸映射（数值唯一来源）

**移动距离（`move.delta` 的标量）**

| 话术 | 像素 |
|---|---|
| 一点 / 一点点 / 稍微 | 60 |
| 一些 / 一段 | 120 |
| 很多 / 一大截 / 远一点 | 240 |
| 显式数字（"100"/"一百"） | 原值（中文数字转阿拉伯） |

**缩放（`resize.scale`）**

| 话术 | factor |
|---|---|
| 大一点 / 变大 / 大一些 | 1.3 |
| 大很多 / 大大的 | 1.8 |
| 大一倍 / 放大一倍 / 两倍 | 2.0 |
| 小一点 / 变小 / 小一些 | 0.77 |
| 小很多 | 0.55 |
| 缩小一半 / 一半 | 0.5 |

**语义尺寸 → 特征尺寸 v（`SizeSpec` 的 "small"/"medium"/"large"）**

| | small | medium（缺省） | large |
|---|---|---|---|
| v | 40 | 80 | 160 |

触发词：小/小小的→small；大/大大的/很大→large；迷你→small；巨大→large×1.5（=240）。

**特征尺寸 v → 各形状几何（引擎换算规则）**

| shape | 换算 |
|---|---|
| circle | 半径 = v |
| ellipse | 长半轴 = v，短半轴 = 0.6v |
| rect（正方形） | 边 = 2v |
| rect（长方形） | 宽 = 2v，高 = 1.5v |
| triangle | 边长 = 2v |
| star | 外半径 = v，内半径 = 0.5v |
| line | 长度 = 3v |
| text | fontSize = max(16, 0.5v) |

**相对尺寸**："比 X 矮/小一点" → `{relativeTo: X, factor: 0.7}`；"和 X 一样大" → factor 1.0；"比 X 大/高" → 1.3。
维度规则：用于 `width` 槽位 → factor × 参照包围盒宽；`height` → factor × 参照高；`size`（特征尺寸）→ factor × max(参照宽, 高) / 2。

**间距**：`gap` 缺省 20（对象间）；ref=canvas 时内边距 40。

**缺省样式**（创建时未指定颜色）：闭合图形 fill `#4B5563`；line/polyline stroke `#111827`、strokeWidth 3；text fill `#111827`。rect 仅给 size 时默认长方形（宽 2v 高 1.5v），正方形由理解层显式产出 width=height。

### 2.5 忽略词（匹配前剔除，不计入未消费 token）

帮我、请、给我、麻烦、来、一下、吧、啊、呢、哦、嗯、那个（句中填充）、然后、就、把（保留语法功能后剔除）。

### 2.6 确认窗口词表（AWAIT_CONFIRM 子态专用）

- 肯定：确认、确定、对、对的、是、是的、好、好的、嗯、可以、行、清空吧、删吧
- 否定：取消、不、不要、不用、别、算了、等等
- 其他任何转写 → 视为否定（保守策略），TTS"已取消"

### 2.7 序数与指代

第一个/第1个→`ordinal:1`，第二个→2……；最后/最后一个/最新的→`"last"`；最早/最先→`"first"`。
它 / 这个 / 那个（句首单独成分）/ 刚才那个 / 刚画的 → `{byFocus:true}`。

## 3. 同音词纠错

### 3.1 流程

```
ASR final → ① 精确词表替换（§3.2，带上下文条件） → ② 剩余未识别 token 拼音回退（§3.3） → 进路由器
```

纠错只发生在规则层匹配之前；送 LLM 的 `utterance` 用纠错后文本，原文放 `asr_alternatives[0]`。

### 3.2 纠错词表（领域核心，约 60 条）

**动词类**（条件：处于动词槽位，即句首或"把…"之后）
花→画、划→画、化→画、桦→画、删→删（保持）、山除→删除、闪出→删除、车销→撤销、撤消→撤销、重作→重做、虫做→重做、青空→清空、轻空→清空、亲空→清空、保存→保存、报存→保存

**图形类**
园→圆、元→圆、源→圆、缘→圆、原→圆（仅后接"形/圈"或量词前）、原形→圆形、原型→圆形、园形→圆形、方快→方块、方亏→方块、长方型→长方形、三脚形→三角形、三角型→三角形、五角星→五角星（保持）、午角星→五角星、星星→星星（保持）、香线→线（"画条香线"场景少，低优先）、值线→直线、知线→直线

**颜色类**
篮色→蓝色、兰色→蓝色、蓝→蓝（保持）、绿色→绿色（保持）、率色→绿色、吕色→绿色、黄色→黄色（保持）、皇色→黄色、汇色→灰色、辉色→灰色、紫色→紫色（保持）、子色→紫色、橙色→橙色（保持）、成色→橙色（条件：颜色槽位）

**方位/操作类**
做边→左边、坐边→左边、右变→右边、伤面→上面、下边→下边（保持）、虾面→下面、中减→中间、放大→放大（保持）、防大→放大、缩小→缩小（保持）、所小→缩小、锁小→缩小、依动→移动、咦动→移动、旋转→旋转（保持）、悬转→旋转

> 词表维护：开发期遇到新 badcase 直接加表；"保持"条目用于测试基线，确认不会被误纠。

### 3.3 拼音回退（词表未命中时）

1. 对纠错后仍无法参与任何模板匹配的 2~3 字 token，求其无声调拼音
2. 与领域词典（§2.1~§2.3 全部词条 + 动词表）逐一比对拼音编辑距离
3. 距离 ≤ 1 且唯一命中 → 替换；多个命中或距离 > 1 → 不替换（留给 LLM）
4. 实现：`pinyin-pro` 库；词典预计算拼音建索引，单次纠错 < 5ms

## 4. 规则模板文法

### 4.1 匹配流程

```
纠错后文本
 → 剔除忽略词（§2.5）
 → 分词 + 槽位标注（最长匹配扫描 §2 各词表，产出 token 流：
    [动词][颜色][尺寸][形状][方位][数量][目标指代][名称]…）
 → 按 T1~T11 依序尝试模板
 → 命中且 未消费 token 占比 ≤ 30% → ParseResult{source:"rule", confidence:1.0}
 → 否则 → 升级 LLM（mode 判定见 §4.3）
```

“未消费 token 占比”按字符数计。阈值 30% 的含义：允许少量口语噪音，但句中存在成段未理解内容时宁可升级 LLM 也不猜。

### 4.2 模板清单 T1~T11

| # | 模板（槽位顺序宽松，`?`=可选） | 产出 Op | 示例 |
|---|---|---|---|
| T1 创建 | 动词(画/添加/加/来/写) + 位置? + 尺寸? + 颜色? + **形状** + 内容?(text 用) + 位置? | `create` | "在左上角画个大的蓝色矩形" |
| T2 移动 | 目标? + 动词(移/挪/移动/拖) + **方向**(上/下/左/右) + 量词? | `move.delta`（方向×量词，缺省 60） | "把圆往右移一点" |
| T3 缩放 | 目标? + (变/放/缩) + **大/小** + 量词? ｜ 放大/缩小 + 倍数? | `resize.scale` | "把它变大一点" |
| T4 改色 | 目标? + (涂成/变成/改成/换成/涂) + **颜色** | `style.fill` | "把房子涂成红色" |
| T5 删除 | **目标** + (删掉/删除/去掉/移除/擦掉) | `delete` | "把那个圆删掉" |
| T6 撤销/重做 | **撤销/撤回/回退** + N步? ｜ **重做/恢复** | `undo/redo` | "撤销三步" |
| T7 清空 | **清空**(画布)? ｜ 全部删掉 ｜ 重新开始/重画 | `clear` → **confirm-pending** | "清空画布" |
| T8 旋转 | 目标? + (转/旋转) + **N 度** ｜ +方向(顺/逆时针)? | `rotate.degrees`（逆=负） | "把三角形转 45 度" |
| T9 命名 | (这个/它) + **叫/命名为** + 名称 | `rename`（目标=焦点） | "这个叫屋顶" |
| T10 导出 | **保存/导出/下载** + 图片? | `export png` | "保存图片" |
| T11 选中 | **选中/选择** + 目标 | `focus` | "选中那个红色的圆" |

**目标槽位的统一解析**（T2~T5、T8、T11 共用）：
1. 指代词（§2.7）→ `byFocus`
2. 已命名对象名 → `byName`（名称表来自场景图，匹配时热更新）
3. `[颜色?] [序数?] [形状]` → `byQuery`
4. 槽位缺失（"变大一点"没说目标）→ `byFocus`；若焦点为空 → 不命中，**不升级 LLM**，直接 TTS"请先告诉我要操作哪个图形"（这是明确的信息缺失，LLM 也解决不了）

**T1 细则**：
- 位置槽位 = 方位词（→ `{ref:"canvas", anchor}`）或相对介词结构"在 X 的 D 边"（→ `{ref:目标, anchor:D}`）。两处位置槽位最多出现一个。
- "画两个圆" → 数量词展开为 2 个 `create`（自动布局错开），上限 5 个，超出升级 LLM。
- text 形状必须有引号内容或"写/字是"引导（"写上你好"→ `text:"你好"`），否则不命中。

### 4.3 升级 LLM 时的 mode 判定

命中以下任一 → `mode:"plan"`，否则 `"parse"`：
- 动词"画"+ 非 §2.2 词表的具体名词（雪人/房子/树/太阳/小猫…即"画的不是基础图形"）
- "一幅 / 一张 / 一个场景 / 风景"
- 句含 2 个以上"然后/再/接着"连接的创建动作

## 5. 执行引擎语义细则

### 5.1 焦点对象（focus）更新规则

| 事件 | 焦点变化 |
|---|---|
| 事务含 `create`（parse/规则来源） | = 最后一个 create 的对象 |
| 事务来源 `llm-plan` | 引擎自动将本事务全部新建对象 `group`（组名 = 用户话术中的主名词，如"雪人"）；焦点 = 该组 |
| `style/move/resize/rotate/setText/rename` | = 被操作对象（多目标取最后一个） |
| `delete` | 焦点清空 |
| `focus` op | 显式设置 |
| `undo/redo` | 恢复为快照中记录的焦点（见 §5.4） |
| `clear` | 焦点清空 |

### 5.2 自动布局（create 缺省 `at` 时）

确定性算法（无随机，保证可测试）：

```
1. 画布为空 → center (512, 384)
2. 有焦点对象 F → 依次尝试 F 的 right / below / left / above（gap=40），
   取第一个与所有现有 bbox 不重叠、且完全在画布内的位置
3. 仍无 → 依次尝试画布九宫格锚点（center, top-left, top, top-right, left,
   right, bottom-left, bottom, bottom-right，内边距 40），同样判重叠
4. 仍无（画布很满）→ 直接 center 叠放，不报错
```

### 5.3 相对定位的精确计算

记 ref 包围盒 `R`，新对象包围盒 `N`（尺寸已由 SizeSpec 确定）。

**ref 为对象 → 外贴（outside）**：

| anchor | N 的位置 |
|---|---|
| `left` | N.right = R.left − gap；N 垂直中心 = R 垂直中心 |
| `right` | N.left = R.right + gap；垂直居中对齐 |
| `top` | N.bottom = R.top − gap；水平居中对齐 |
| `bottom` | N.top = R.bottom + gap；水平居中对齐 |
| 四角（如 `top-left`） | N 的 bottom-right 角 = R 的 top-left 角 + (−gap, −gap) |
| `center` | N 中心 = R 中心（叠放，画雪人眼睛等场景） |

**ref 为 canvas → 内贴（inside）**：N 贴画布对应边/角，内边距 40；`center` = 画布中心。

`offset` 在上述结果上叠加。最终位置再过 §5.5 clamp。

### 5.4 undo/redo：快照制

- 不做逆操作（command inverse），**每事务结束后保存整幅场景 JSON 快照**（含焦点 id）。画布对象数量级 ≤ 百，快照成本可忽略，换取绝对正确性
- 栈深 50，超出丢最旧；redo 栈在新事务提交时清空
- `undo steps:n` = 弹 n 帧；n 超过可用深度时撤到栈底为止、不报错（栈本来就空才报 NOTHING_TO_UNDO）
- `clear` 也入栈（可被撤销——这是清空敢于轻确认的底气）；clear 不重置 id 计数（保证 id 全程不复用）
- 事务部分失败（executed > 0）时同样入栈：已生效的部分必须可撤销

### 5.5 越界 clamp

1. bbox 超出画布 → 平移最小距离使其完全入界
2. 对象本身大于画布（平移无解）→ 等比缩小至画布的 90% 后居中
3. clamp 发生时不报错、不播报，仅事件总线发 `exec.clamped`（调试面板可见）

### 5.6 group 的引用语义

- **几何类 Op**（move/resize/rotate/delete/zorder）：target 命中组内成员时**提升为整组**
- **外观类 Op**（style/setText/rename）：作用于**成员本身**（"把雪人的鼻子涂红"改的是鼻子）
- byQuery 在组内外都有命中时：优先组外独立对象；仅组内命中则按上述规则
- `ungroup` 后成员保留各自 id 与 name；组名作废

### 5.7 歧义处理（AMBIGUOUS_TARGET）

- 候选 ≤ 3：TTS 列举区分特征——优先用颜色区分（"有红色和蓝色两个圆，要哪个？"），颜色相同用位置（"左边的还是右边的？"），并把候选写入 `clarify.expecting`
- 候选 > 3：TTS"画布上有 N 个圆，请说得具体一点，比如颜色或位置"
- 澄清回答的快匹配：下一句 final 先与 `expecting` 做包含匹配，命中则直接补全原 Op 执行，**不再走完整解析**

## 附录 A：LLM System Prompt 全文（构建期由 lexicon 生成，运行期逐字节不变）

````text
你是一个语音绘图工具的指令解析器。用户通过语音下达绘图指令，你把中文口语转换成 JSON 绘图操作。

# 输出格式
你必须只输出一个 JSON 对象，不要输出任何其他文字：
{
  "intent": "ops" | "clarify" | "reject",
  "confidence": 0到1的数字,
  "ops": [Op数组，intent为ops时非空，否则为空数组],
  "say": "播报给用户的话，不超过20个字",
  "clarify": {"question": "...", "expecting": ["候选1","候选2"]}  // 仅intent=clarify时
}

# 绘图操作 Op 速查
create: {op,shape,name?,at?,size?,width?,height?,points?,text?,fill?,stroke?,rotation?,desc?}
  shape: circle|ellipse|rect|triangle|line|polyline|star|text
style:  {op,target,fill?,stroke?,strokeWidth?,opacity?}
move:   {op,target,to?|delta?[dx,dy]}        resize: {op,target,scale?|to?}
rotate: {op,target,degrees}                  delete: {op,target}
rename: {op,target,name}                     setText:{op,target,text}
group:  {op,targets[],name?}                 zorder: {op,target,to:front|back|forward|backward}
focus:  {op,target}
禁止输出 clear、undo、redo、export（这些由本地处理）。

# target 选择器（按优先级使用）
{"byName":"屋顶"} > {"byFocus":true}（用户说"它/刚才那个"） > {"byQuery":{"shape":"circle","fill":"#FF4136","ordinal":"last"}}
scene.objects 里有当前画布所有对象；focusId 是焦点对象。引用对象前先在 scene 里确认它存在。

# 位置 at（声明式，不要自己算绝对坐标，除非必要）
绝对: {"x":512,"y":384}（画布1024×768，左上为原点）
相对: {"ref":"canvas"或target选择器, "anchor":"left|right|top|bottom|center|top-left|...", "gap":20, "offset":[dx,dy]}
ref是对象时为外贴（"在房子左边"），ref是canvas时为内贴（"在画布左边"）。

# 尺寸 size
数字=特征尺寸（圆为半径）；"small"=40 "medium"=80 "large"=160；
相对: {"relativeTo":目标,"factor":0.7}（"比它矮一点"=0.7，"一样大"=1.0，"比它大"=1.3）
相对尺寸维度：width=factor×参照宽 height=factor×参照高 size=factor×max(参照宽,高)/2

# 数值映射（必须使用这些值）
移动"一点"=60px "一些"=120px "很多"=240px
缩放"大一点"=1.3 "大很多"=1.8 "一倍"=2.0 "小一点"=0.77 "一半"=0.5
颜色用十六进制：红#FF4136 蓝#0074D9 天蓝#87CEEB 黄#FFDC00 金#FFD700 绿#2ECC40
橙#FF851B 紫#B10DC9 粉#FFB6C1 棕#8B4513 黑#111111 白#FFFFFF 灰#AAAAAA 青#39CCCC

# 规则
1. 用户话语可能含语音识别错误，参考 asr_alternatives 推断本意（"花一个园"="画一个圆"）。
2. 指令有歧义且 scene 无法消解时，用 intent=clarify 提问，expecting 列出候选答案。
3. 与绘图无关的请求（聊天、问天气）用 intent=reject。
4. 没把握时降低 confidence；低于 0.6 系统会自动转为向用户确认。
5. mode=plan 时（创作拆解）：把图案拆为 5~20 个 create Op，从背景到前景、从大到小；
   每个 Op 必须带 desc（如"画雪人的头"）；部件间用相对定位（ref 上一个部件的 name）
   保证整体协调；给关键部件起 name。
6. say 要口语化、简短：“好了，画了一棵树”而不是“已成功执行创建操作”。
````
